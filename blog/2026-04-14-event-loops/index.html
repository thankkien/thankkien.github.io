Vietnamese

Event Loop

Event Loop là cơ chế cho phép xử lý các tác vụ bất đồng bộ trong JavaScript, sử dụng một vòng lặp vô tận để kiểm tra và quản lý các tác vụ. JavaScript có thể xử lý nhiều tác vụ cùng lúc mặc dù JavaScript là single-threaded — tại một thời điểm chỉ thực hiện một tác vụ duy nhất.

Call Stack

  • Mỗi khi một hàm được gọi, nó được đưa vào Call Stack và chờ được thực thi. Khi hàm thực thi xong, nó sẽ được đẩy ra khỏi Call Stack.
  • Thứ tự: FILO (First In, Last Out) — hàm được gọi trước thì thực thi sau.
  • Ví dụ: Call Stack có 2 tác vụ longRunningTask() và importantTask() => dù importantTask() có quan trọng đến đâu thì cũng phải đợi longRunningTask() chạy xong.

Web APIs (Node APIs in Node.js)

  • Các hàm bất đồng bộ như setTimeout, fetch, DOM events, etc.
  • Khi Call Stack gặp một hàm bất đồng bộ, nó được offload ra khỏi stack để Web APIs xử lý.

Queue

  • Sau khi các hàm đồng bộ xử lý xong, callback sẽ được đưa vào queue.
  • Task Queue (Macrotask Queue): chứa callback từ các Web APIs như setTimeout, DOM events, etc.
  • Microtask Queue: chứa callback từ các hàm Promise-based như Promise.then, async/await, etc. và có độ ưu tiên cao hơn Task Queue.

Event Loop (Xử lý)

  • Kiểm tra xem Call Stack có trống hay không.
  • Nếu Call Stack trống, kiểm tra Microtask Queue và thực thi tất cả các callback từ đây.
  • Khi Microtask Queue đã trống, kiểm tra Task Queue và thực thi một callback tại một thời điểm.
  • Quá trình này lặp lại liên tục.

English

Event Loop

The event loop is a mechanism that allows JavaScript to handle asynchronous operations. It uses an infinite loop to continuously check and manage tasks. Even though JavaScript is single-threaded — meaning it can execute only one task at a time—it can still handle multiple operations concurrently thanks to the event loop.

Call Stack

  • Whenever a function is called, it is pushed into the Call Stack and waits to be executed. After execution, it is removed from the Call Stack.
  • The execution order follows FILO (First In, Last Out) — the function called first will be executed last.
  • For example, if the Call Stack has two tasks: longRunningTask() and importantTask(), even if importantTask() is more critical, it still has to wait until longRunningTask() finishes.

Web APIs (Node APIs in Node.js)

  • Asynchronous functions such as setTimeout, fetch, and DOM events are handled by Web APIs.
  • When the Call Stack encounters an asynchronous function, it is offloaded from the stack and handled by these APIs.

Queue

  • After synchronous tasks are completed, callbacks are placed into queues.
  • Task Queue (Macrotask Queue): contains callbacks from Web APIs like setTimeout, DOM events, etc.
  • Microtask Queue: contains callbacks from Promise-based operations such as Promise.then, async/await, etc. and has higher priority than the Task Queue.

Event Loop (Process)

  • The event loop continuously checks whether the Call Stack is empty.
  • If the Call Stack is empty, it first checks the Microtask Queue and executes all callbacks from it.
  • Once the Microtask Queue is empty, it checks the Task Queue and executes one callback at a time.
  • This process repeats continuously.
